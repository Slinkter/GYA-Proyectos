## REGLAS DE CГ“DIGO

### TypeScript

- NUNCA uses `any` вҖ” usa `unknown` si es necesario y luego narrowing
- Todos los props de componentes deben tener interface explГӯcita
- Importar tipos del backend desde shared/types (definidos en SCHEMA.md)

### Componentes React

- SIEMPRE 'use client' en componentes con estado o efectos
- Server Components para pГЎginas que solo leen datos
- NUNCA lГіgica de negocio en componentes вҖ” va en hooks

### Tailwind CSS v4

- Variables CSS en app/globals.css:

  --color-gna-red: #CC0000;
  --color-gna-black: #000000;
  --color-gna-gray: #F7F7F7;

- NUNCA Inter, Roboto o fuentes genГ©ricas para el documento imprimible
  вҶ’ Times New Roman para cuerpo, Arial para headers (como el original)

### Formulario de Presupuesto вҖ” ItemsTable

Cada fila tiene EXACTAMENTE estos campos en una sola lГӯnea:
| PRODUCTO | SISTEMA | DESCRIPCIГ“N | ALTURA | ANCHO | PRECIO |
| (input) | (input) | (textarea) | (num) | (num) | (num) |

- Campos libres: el usuario conoce precios de memoria
- NO hay tabla de precios predefinida
- MГӯnimo 1 fila al cargar, botГіn para agregar mГЎs
- BotГіn eliminar fila (Гӯcono trash) al final de cada fila

### Llamadas a la API

- Usar axios desde shared/lib/axios.ts (ya tiene baseURL y token interceptor)
- Usar TanStack Query para GET (cache automГЎtico)
- Usar mutations para POST/PUT/DELETE
- NUNCA fetch() directo вҖ” siempre a travГ©s de axios configurado

### GeneraciГіn SVG

- El usuario describe el producto en lenguaje natural
  Ejemplo: "Ventana corrediza 2 hojas, ancho 1200, alto 900, sistema Nova, vidrio 6mm"
- Frontend envГӯa POST /api/svg/generate con { descripcion: string }
- Backend responde { svgCode: string }
- Frontend muestra preview con dangerouslySetInnerHTML
- Usuario aprueba вҶ’ SVG se agrega al array de diagramas del presupuesto

### ImpresiГіn / PDF

- Usar react-to-print con ref al contenedor de impresiГіn
- @media print en globals.css:

- PГЎgina 1: DocumentoPagina1 (texto, tabla productos, condiciones, firmas)
